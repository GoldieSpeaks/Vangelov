Client Analytics Dashboard
Designing and delivering a self-service Payment Dashboard for API-integrated clients across the education payments ecosystem, closing a critical competitive gap.

Prior to this initiative, API clients could only access payment data programmatically through API endpoints. They had no visual analytics, reporting tools, or self-service dashboard.

The new dashboard centralised transaction tracking, added multi-currency insights, introduced exportable reports for reconciliation, and enabled client account management through a phased delivery approach.

Confidentiality Notice: Internal schemas, client identifiers, and sensitive operational details are generalized while preserving the delivery approach and product outcomes.

Project Scope

In Scope

  • Payment dashboard for API clients with analytics and reporting
  • Multi-currency data visualisation and transaction tracking
  • Master and sub-account setup with access controls

  • Security validation and penetration testing
  • RAID log and privacy impact assessment support

Out of Scope

  • Disbursement tracking in initial MVP release
  • White-label feature pack in initial MVP release

Lessons Learned
What Worked

Phased MVP Delivery Reduced Risk

Launching early enabled real-world feedback before heavy design investment, improving the quality of later iterations.

Key Insight

Security Validation Built Confidence

External penetration testing provided assurance for both compliance and stakeholder trust ahead of full adoption.

Watchout

Scope Expansion Needs Tight Governance

Post-MVP enhancements can grow rapidly; separate BRD cycles and sequencing were essential to maintain control.

Improvement Area

Earlier Data Model Alignment Helps

Aligning back-office structures and client-facing account models sooner would have reduced integration complexity.
