Platform Infrastructure Migration & Language Upgrade

E Executive Summary

The cross-border payments platform was running on legacy infrastructure with an outdated server-side language runtime. A pre-existing cloud services contract was underutilised, and the current hosting arrangement did not deliver the scalability, security, and cost efficiency the platform required as transaction volumes grew.


This project delivered a dual-scope migration: moving all web and API services to cloud infrastructure while simultaneously upgrading the server-side runtime to a modern, supported version. The initiative was classified as high priority, causing multiple other projects to be deferred until completion. With a large team spanning development, QA, business analysis, and infrastructure, the project progressed through iterative development cycles with multiple release target adjustments before successful production deployment.

💡 Lessons & Insights

What Worked Well

Bundling Migration with Runtime Upgrade

Combining both initiatives into a single project avoided double the testing effort and double the production cutover risk. The team only needed one major stabilisation period rather than two.

What Worked Well

Full Team Dedication

By pausing all other projects, the full team was focused on the migration. This prevented context-switching overhead and ensured the most critical initiative received undivided attention.

Challenge

Four Release Target Movements

The release date moved four times as the true scope of runtime compatibility issues became apparent. While each move was justified, it created planning uncertainty for dependent projects waiting in the queue.

Impact

Cascading Project Delays

Multiple other projects — including the API payment instructions rework — were deferred for several months. This created a backlog of delayed initiatives that required careful re-prioritisation after migration completed.

Key Learning

Scope Discovery Is Inherent in Infrastructure Projects

Runtime upgrade compatibility issues cannot be fully enumerated in advance. Each testing cycle revealed new breakages, requiring new deliverables. Future infrastructure projects should budget for at least 30% scope growth.

Key Learning

Coordinator Role Critical for Large Teams

Reassigning the business analyst to an active coordination role mid-project significantly improved cross-stream alignment. For projects with 8+ team members, a dedicated coordinator should be assigned from day one.
