Alternative Payment Method Provider Migration

Migrating alternative payment methods from a high-cost processor to a more cost-effective regional provider — expanding payment options while reducing transaction fees for cross-border payments from China.

E Executive Summary

Cross-border payments from China were processed through a global payment processor whose fee increases had made alternative payment method (APM) transactions unprofitable. The platform needed to migrate these payment methods to a regional provider specialising in Chinese cross-border payments, which offers lower transaction fees and supports additional payment methods not previously available.


The migration involves redirecting WeChat Pay, Alipay, and UnionPay transactions from the current global processor to the regional provider's API. This adds new payment methods (WeChat Pay, UnionPay Debit, UnionPay Credit) that were not previously supported, and introduces category-specific payment rules based on tuition, accommodation, and other payment types. The project also involves regulatory compliance — Chinese payment regulations require payer documentation to be collected through the provider.


The project has been released to production and is now live. The full scope — payment method configuration, exception rules, provider API integration, and category-specific payment logic — has been delivered and is operational.

☑ Project Scope

In Scope

  • Migration of APMs from global processor to regional provider
  • New payment methods: WeChat Pay, UnionPay Debit, UnionPay Credit
  • Category-specific payment rules (tuition, accommodation, etc.)
  • Payment method exception engine by category and link
  • Provider API integration with redirect flow
  • Regulatory document collection via provider portal
  • Web platform and API system integration

Out of Scope

  • Agent payment flows
  • Invoice API payment types
  • Non-China countries using the same processor
  • Card payments (Visa/MasterCard) via existing processor
  • Company-type payment configurations

⚙ Development Areas

Payment Method Configuration

Setting up new payment methods in the platform database with country limitations, role-based access rules, and provider linkage. Each method maps to specific payment categories with distinct fee structures.

Exception Engine

Complex exception logic determining which payment methods are available based on payment category (tuition vs accommodation), accommodation flags, specific link IDs, and exception-to-exception rules for edge cases.

Provider API Integration

New integration flow redirecting payers to the regional provider for payment completion. Includes signature generation, request/response handling, webhook processing, and parameter mapping between platform database and provider API.

Regulatory Compliance

Chinese payment regulations require payer documentation (ID verification, proof of payment purpose). The provider handles document collection via a time-limited portal link, removing this burden from the platform.

★ Lessons & Insights

Design Decision

Category-Based Payment Rules

Rather than a simple per-country payment method list, the BRD defines payment availability by payment category (tuition, accommodation, blocked accounts, etc.). This granular approach reflects real-world regulatory and provider constraints but adds significant complexity to the exception engine.

What Worked Well

Comprehensive BRD Coverage

The BRD covered payment method configuration, exception logic, API integration details, parameter mapping, and use cases. This level of documentation thoroughness reduced ambiguity during development and testing.

Consideration

Exception Complexity

The exception engine involves accommodation flags, link ID arrays, exception-to-exception overrides, and category-based visibility rules. This multi-layered logic will require extensive test coverage to ensure correct payment method display across all scenarios.

Risk

Provider Dependency for Document Collection

Regulatory compliance for Chinese payments depends on the provider's document collection portal. Any changes to the provider's document requirements or portal availability directly impact the payment flow and are outside platform control.
