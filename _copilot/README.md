# Copilot Working Context

This folder stores persistent project context for the portfolio transformation.

## Purpose
- Keep execution rules and quality standards.
- Maintain the current website map.
- Keep NDA-safe case study writing framework.
- Track the active backlog and priorities.
- Standardize how project case studies are reworked into portfolio-ready pages.

## Files
- `RULES_AND_GUIDELINES.md` — hard rules for all edits.
- `SITE_MAP.md` — page inventory and architecture map.
- `NDA_CASE_STUDY_FRAMEWORK.md` — writing structure for safe/credible case studies.
- `WORKING_BACKLOG.md` — short-term execution plan.

## Case Study Direction
- Rework many project pages into a consistent premium case-study format.
- Use the strongest approved project pages as style references before inventing new layouts.
- Preserve the portfolio shell: sidebar navigation, shared CSS/JS, and overall visual language.
- Transfer source case study content faithfully, then reshape it into clearer sections, stronger hierarchy, and better visual storytelling.
- Keep every page NDA-safe while still concrete, credible, and outcome-oriented.
- When a temporary build is approved, promote it into the canonical `Projects/*.html` page and remove the temporary project file.

## Update Protocol
After each significant change:
1. Update `SITE_MAP.md` if page structure/navigation changed.
2. Update `WORKING_BACKLOG.md` statuses.
3. Keep all case study copy aligned with `NDA_CASE_STUDY_FRAMEWORK.md`.
