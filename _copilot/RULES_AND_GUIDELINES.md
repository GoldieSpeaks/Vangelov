# Portfolio Project Rules & Guidelines

## Project North Star
Build a job-winning product portfolio that demonstrates:
- Strategic product ownership.
- Measurable business impact.
- Strong UX and delivery thinking.
- Clear communication for recruiters and hiring managers.

## Editorial Rules (Global)
1. Every project page must answer: problem, role, decisions, impact.
2. Prefer evidence over adjectives.
3. Use concise language and scannable sections.
4. Keep tense consistent (past tense for completed work).
5. Remove placeholder/template text.
6. Eliminate outdated references (generic template names, fake dates, fake comments).

## NDA-Safe Communication Rules
1. Never expose confidential client details, internal tooling names, or exact architecture if private.
2. Use ranges or percentage deltas where needed.
3. Replace sensitive proper nouns with safe labels when required (e.g., "Global Education Client", "Enterprise Partner A").
4. Focus on your method and outcomes, not proprietary implementation details.
5. Use defensible metrics only (real or safely abstracted).

## Case Study Quality Bar
Each case study must include:
- Context snapshot (domain, scale, constraints).
- Your explicit role and ownership boundaries.
- Baseline state and key challenges.
- Approach and trade-offs.
- Delivery and validation process.
- Outcome metrics (before/after or directional impact).
- Key learnings and next-step opportunities.

## UX/Content Rules
- Headline should state business value, not only feature name.
- First viewport must contain role + impact summary.
- Use bullets/tables for metrics and decisions.
- Keep paragraphs short (2-4 lines max).

## Technical Rules
- Preserve current visual style while improving structure and clarity.
- Reuse shared CSS/JS where possible.
- Keep navigation consistent across pages.
- Prefer fixing root content issues before visual embellishments.

## Prioritization Logic
1. High-traffic pages first (`index.html`, `all-projects.html`).
2. Core proof pages second (`Projects/*.html` case studies).
3. Supporting pages third (`partnerships.html`, `certification.html`, utilities).
4. Archive/test pages last.

## Definition of Done (Per Page)
- No placeholder lorem text.
- No broken internal links.
- Clear role + impact visible.
- NDA-safe language verified.
- Page contributes to hiring narrative.
