# NDA-Friendly Case Study Framework

Use this structure for every project page.

## 1) Header Block (Above the Fold)
- Project label (safe/client-approved naming).
- Your role.
- Timeline.
- Domain (e.g., cross-border payments, education fintech).
- Quick impact line (1 sentence).

Template:
> Led [initiative] as [role], improving [key KPI] by [delta/range] while operating within [constraint].

## 2) Context
Answer:
- What business problem existed?
- Who was impacted?
- Why did this matter now?

Keep it concise: 1-2 short paragraphs.

## 3) Constraints (NDA-safe)
List constraints without private details:
- Regulatory/compliance boundaries.
- Legacy platform limitations.
- Tight deadlines.
- Multi-stakeholder alignment.

## 4) Ownership & Responsibilities
Use explicit bullets:
- Owned discovery and problem framing.
- Prioritized roadmap and milestones.
- Coordinated design + engineering + operations.
- Set KPI definitions and validation loop.

## 5) Approach
Split into phases:
1. Discovery
2. Strategy
3. Delivery
4. Validation

Each phase should include actions + rationale.

## 6) Outcomes (Proof)
Always include measurable outcomes where possible:
- Conversion/completion change.
- Time-to-complete change.
- Support-volume change.
- Operational/revenue impact.

NDA-safe metric patterns:
- Percent delta: "+18% completion"
- Range: "reduced handling time by 30–40%"
- Indexed baseline: "from baseline index 100 to 128"

## 7) Learnings
Show seniority with reflection:
- What trade-off was hardest?
- What would you do earlier next time?
- What system/process improvement did you institutionalize?

## 8) Reusable Copy Rules
- Avoid confidential architecture diagrams unless approved.
- Avoid exact volume/revenue unless publicly sharable.
- Prefer "customer outcomes + business outcomes" pairings.
- Keep each section skimmable (bullets + short paragraphs).

## 9) Minimum Acceptance Checklist
Before publishing a case study:
- [ ] Role is explicit.
- [ ] Problem is concrete.
- [ ] Approach is structured.
- [ ] Outcome has measurable signal.
- [ ] NDA-sensitive details removed.
- [ ] Spelling/grammar verified.
