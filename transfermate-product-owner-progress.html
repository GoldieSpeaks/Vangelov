Daily Cycle AI-Augmented

Every day follows a structured cycle: rough notes from 5–10 calls → AI processing → structured Jira tasks, Confluence updates, dashboard refresh, action items, decisions logged. The "PO End" sequence is an 11-step shutdown that touches every system.

  • Transcript enrichment from .docx files (Bulgarian + English)
  • Automatic action item cross-referencing across the week
  • Session handoff files for seamless continuity
  • Image processing via vision (diagrams, screenshots)

PO End Sequence (11 Steps)

The full shutdown: Scan recordings → Read transcripts → Enrich notes → Dashboard update (every section) → KB update → Daily summary → Approval report (Confluence + Jira proposals) → Execute approved changes → Verbal summary → Session handoff → Archive + image cleanup.

Weekly Sync System

Auto-generated weekly sync files with WEB and CRM tables for team calls. Claude suggests topics based on the week's activity; I promote them to the main tables. Dashboard renders suggestions below the WEB/CRM panels. Decisions older than 2 weeks auto-archive.

Operational Sequences

Three documented, repeatable sequences that run on command: JSM Create (per-release holder tasks), Missing Time Audit (quarterly Excel report), and Quarter Wrap-Up (3-phase audit + Confluence report for stakeholders).

Jira Sync Auto

Pulls unreleased versions from Jira every 60 minutes. Shows issue status breakdowns per release, today's movements, and flagged impediments. Writes to jira-releases.json.

Overdue Check Auto

Scans DASHBOARD.md action items and flags any older than 7 days. Parses source dates, detects priority column format, and skips struck-through items. Writes to overdue-actions.json.

Reminder Check Auto

Scans daily notes and memory files for date-tagged reminders (deadlines, go-lives, follow-ups). Categorises as today, upcoming, or past. Writes to reminders.json.

2022 — Joined as Product Owner

Took ownership of the WEB payment platform and began learning the domain — cross-border education payments, currency conversion, regulatory requirements. Inherited a basic Jira project with minimal structure.

Created the TRA space from scratch — designed the page hierarchy, authored the first BRDs, set up release note archives, partner pages, and project trackers. Established Confluence as the team's single source of truth.

2023 — Restructured Jira

Introduced ADF task templates, naming conventions, fix version management, and system labels. Transformed Jira from a flat task list into a structured, traceable project management system.

2023 — Took On the CRM Platform

Expanded from WEB-only to dual-platform ownership. Began managing CRM projects: bond automation, compliance screening, Deutsche Bank integration, agent management. Now the sole PO for both systems.

2024 — Set Up JSM Ticket System

Configured Jira Service Management for the team — request types, notifications, access control. Created the release-linked JSM holder task convention.
