5 Overview
Executive Summary

I identified that the platform held personal data for inactive accounts across eight distinct categories, each with different data structures and retention requirements. Under GDPR, this data had exceeded its lawful retention period and needed to be permanently deleted with full documentation to demonstrate compliance during potential regulatory audits.

I designed a phased deletion programme, processing each account type individually: I defined the deletion criteria, coordinated script development, verified test environment execution, oversaw production execution, and ensured record counts were documented. I obtained executive sign-off before production deletion began, ensuring senior leadership endorsed the scope and approach.

I successfully delivered all eight account types through the pipeline, with verified deletion logs documenting the exact records removed. The resulting compliance package was subsequently used to demonstrate adherence during a regulatory review.

12 Learnings
Lessons & Insights
● What Worked Well

Phased Account-by-Account Approach

Processing each account type independently meant that issues discovered in one type (such as edge cases requiring script corrections) did not block progress on other types. The phased approach also provided natural checkpoints for stakeholder communication.

● What Worked Well

Executive Sign-Off Process

Obtaining formal senior leadership approval before any irreversible production action created clear organisational accountability. When the audit documentation was later needed for regulatory review, the executive endorsement strengthened the compliance narrative significantly.

● Challenge

Edge Cases in Complex Account Types

Some account types had data relationships that were not immediately apparent from the initial data mapping. Script corrections were needed mid-execution, requiring re-validation through the full pipeline. Earlier deep-dive analysis of data dependencies would have reduced rework.

● Key Learning

Proactive Audit Documentation

Assembling the compliance package immediately after completion — rather than waiting for a regulatory request — meant the organisation could respond to a subsequent audit within hours rather than weeks. This proactive approach should be standard for all compliance-critical projects.

● Risk Managed

Irreversible Production Deletions

Production data deletion is permanently irreversible. The dual-environment validation, executive sign-off gate, per-type isolation, and documented record counts created multiple safety layers that prevented accidental deletion of data that should have been retained.

● Reflection

What I'd Do Differently

Would have pushed for automated deletion scheduling from the start rather than a manual annual exercise. Building the deletion scripts as a reusable automated pipeline with configurable retention periods would have reduced future effort and eliminated the risk of retention deadlines being missed between annual runs.
